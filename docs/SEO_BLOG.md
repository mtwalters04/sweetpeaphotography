# SEO & Blog

A custom blog, hand-tuned for SEO. Not a CMS port.

---

## Storage decision

Two reasonable options:

1. **MDX in repo** (`content/blog/*.mdx`). Fast, version-controlled, trivial to render. Authoring requires a deploy.
2. **Supabase-backed** with a small admin editor (Tiptap or Plate). Authoring without deploys; richer admin.

**Recommendation: Supabase-backed.** The admin already exists; let posts flow through it like everything else. Editing typos shouldn't require Vercel builds. Tradeoff: more code than MDX, and you must care about preview rendering.

## Routes

- `/journal` — index, paginated, filter by tag.
- `/journal/[slug]` — single post.
- `/journal/tag/[tag]` — tag archive.
- `/sitemap.xml` — generated at request time, includes posts + portfolio + service pages.
- `/rss.xml` — full content RSS.

The user-facing word is **Journal**, not "Blog." Quieter.

---

## SEO checklist (per post)

Generated by the system or required by the editor:

- `<title>` — `meta_title || title` + " — Sweet Pea Photography"
- `<meta name="description">` — `meta_description || dek`
- Canonical link
- OpenGraph + Twitter card meta
- JSON-LD `Article` schema with `author`, `datePublished`, `dateModified`, `image`, `publisher`
- JSON-LD `LocalBusiness` schema in the site shell (footer/layout)
- Hero image preloaded with `fetchpriority=high`
- Internal links: every post must link to at least one service page (`session_types.slug`)
- All images have alt text — required field in the editor
- Reading time computed and shown
- Last-updated date shown when `updated_at > published_at + 24h`

## OG images

Generated on demand via `next/og` at the edge:
- Cached in R2 at `/og/{slug}.png`
- Regenerated when post `updated_at` advances
- Template: hero crop + serif title + small-caps brand mark, palette tokens

---

## Sitemap & indexing

- `sitemap.xml` includes: home, all service pages, all `published` blog posts, portfolio pages.
- `robots.txt` allows all, references sitemap, disallows `/admin` and `/account`.
- `lastmod` per URL — pull from `updated_at`.

---

## Local SEO (single market)

- LocalBusiness JSON-LD in root layout.
- Cite the city/region in:
  - Home hero or sub-hero
  - Footer address block (or service area if no public address)
  - At least one h2 on the home page
- Build at least one cornerstone post per service per year that names the city ("Wedding photography in {City}: a 2026 guide").

---

## Performance budgets

These matter for SEO too. Targets:

- LCP ≤ 2.0s on 4G
- CLS ≤ 0.05
- INP ≤ 200ms
- JS shipped to home < 120KB gzipped

If a feature blows the budget, push back before shipping.

---

## What the AI tends to do that hurts SEO here

- Auto-generating slugs that change when titles edit → **slugs are manual, immutable after publish**.
- Adding a "share" widget that injects 200KB of script → don't.
- Stuffing keywords into alt text → alt text describes the image, period.
- Forgetting JSON-LD when scaffolding the post template → it's required, not optional.
